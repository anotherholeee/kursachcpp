# Заключение

В рамках курсового проекта была спроектирована и реализована комплексная система управления городским общественным транспортом, решающая задачи автоматизации процессов планирования и контроля движения транспортных средств. Разработка выполнена на языке C++ с применением стандартной библиотеки STL и объектно-ориентированного подхода, что обеспечило создание масштабируемого и поддерживаемого программного решения.

Центральным компонентом системы является модуль управления расписанием движения, который интегрирует работу с маршрутами, рейсами, транспортными средствами, остановками и водителями в единую информационную среду. Реализованный функционал охватывает полный жизненный цикл данных: от создания и модификации сущностей до их удаления, с поддержкой расширенного поиска, фильтрации по различным критериям и сортировки результатов для удобной навигации по большим объемам информации.

Технической особенностью проекта является реализация набора специализированных алгоритмов для решения задач транспортной логистики. Алгоритм расчета времени прибытия транспорта на остановки использует физическую модель движения с учетом расстояний между остановками, средней скорости транспортного средства и времени стоянки, что позволяет автоматически генерировать точные расписания для произвольных рейсов. Алгоритм создания маршрутов включает многоуровневую валидацию входных данных, проверку логической целостности (исключение циклических маршрутов с одинаковыми начальной и конечной остановками) и контроль уникальности идентификаторов для предотвращения конфликтов в данных.

Особое место в архитектуре системы занимает алгоритм планирования поездок с пересадками, реализованный на основе графовой модели транспортной сети и алгоритма поиска в ширину (BFS). Данный подход позволяет находить оптимальные маршруты между произвольными остановками с минимальным количеством пересадок, строя граф связности остановок через маршруты и выполняя эффективный поиск кратчайших путей. Дополнительно реализованы стратегии поиска самого быстрого маршрута и маршрута с минимальными пересадками, инкапсулированные через паттерн проектирования Strategy для обеспечения расширяемости системы.

Архитектурная организация кода базируется на трехуровневой иерархии классов для представления транспортных средств: базовый класс Vehicle, промежуточные классы FuelTransport и ElectricTransport, и специализированные классы Bus, Trolleybus и Tram. Такая структура обеспечивает корректное моделирование различных типов транспорта с их специфическими характеристиками (тип топлива, напряжение питания, вместимость) и позволяет использовать полиморфизм для унифицированной обработки разнородных объектов. Система обработки исключений гарантирует надежность работы при некорректных входных данных или нарушении бизнес-правил, а механизм валидации обеспечивает целостность хранимой информации.

Функциональность отмены и повтора операций (undo/redo) реализована через паттерн Command, что позволяет отслеживать историю изменений для всех категорий данных и обеспечивает возможность отката до 100 последних операций. Система управления расписанием водителей включает контроль рабочего времени с проверкой соответствия трудовому законодательству, ограничивая максимальную продолжительность рабочего дня 12 часами. Персистентность данных обеспечивается через сериализацию в связанные текстовые файлы с автоматическим сохранением при завершении работы и загрузкой при старте приложения.

Проведенное тестирование подтвердило корректность работы всех компонентов системы: алгоритмов расчета расписаний и поиска маршрутов, механизмов управления данными, операций отмены и повтора действий, а также процедур сохранения и загрузки данных. Разработанное решение соответствует техническим требованиям и готово к использованию в транспортных предприятиях для автоматизации процессов управления расписанием движения городского общественного транспорта.

Реализованная система демонстрирует применение современных подходов к проектированию программного обеспечения на C++ и может служить основой для дальнейшего развития в направлении интеграции с системами мониторинга транспорта в реальном времени, анализа пассажиропотоков и оптимизации маршрутной сети с использованием методов машинного обучения.
