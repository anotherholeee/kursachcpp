# Описание курсового проекта

Курсовой проект реализован в виде программного комплекса, предназначенного для автоматизации процессов управления расписанием городского общественного транспорта. Полный листинг исходного кода размещен в приложении Д.

После запуска приложения открывается интерфейсное окно, оформленное в темных тонах (рисунок 4.1). 

Система построена на основе вкладок, которые обеспечивают быстрый переход между функциональными модулями. Интерфейс организован таким образом, что работа с различными разделами не требует дополнительных действий от пользователя. Программный комплекс функционирует в двух режимах доступа: Гостевой и Администратор. 

В главном окне представлены все ключевые модули системы, организованные в виде вкладок: управление маршрутами, рейсами, транспортными средствами, остановками, водителями, просмотр расписания и поиск маршрутов. Каждая вкладка содержит специализированный интерфейс для работы с соответствующими данными. В верхней части окна расположена панель инструментов с кнопками быстрого доступа: Отменить, Повторить, Сохранить, Загрузить и Обновить.

Программа обеспечивает полноценную работу с различными сущностями системы: для каждой из них (маршруты, рейсы, транспортные средства, остановки, водители) реализованы операции добавления, просмотра, редактирования и удаления (рисунок 4.2). Все данные отображаются в удобных таблицах с возможностью сортировки и цветовой индикации. На рисунке 4.3 продемонстрирован вывод всех маршрутов системы.

Дополнительно в системе организована отмена и повтор последних действий (undo/redo) для каждой категории данных, позволяющая пользователю легко исправить случайные ошибки. На панели инструментов расположены кнопки "Отменить" и "Повторить", которые обеспечивают навигацию по истории выполненных операций. Система поддерживает до 100 последних операций, что позволяет откатить значительное количество изменений. На рисунке 4.4 продемонстрирована отмена добавления маршрута в систему.

Ключевыми алгоритмами программы являются алгоритм расчета времени прибытия транспортного средства на остановки, который позволяет автоматически вычислять расписание движения для любого рейса на основе средней скорости транспорта, расстояния между остановками и времени стоянки на каждой остановке (рисунок 4.5). Пользователь может указать идентификатор рейса и среднюю скорость движения, и система автоматически рассчитает время прибытия на каждую остановку маршрута, учитывая последовательность остановок и время в пути между ними. Алгоритм последовательно обрабатывает каждую пару соседних остановок, вычисляет время в пути на основе расстояния и скорости, определяет время прибытия и учитывает время стоянки для расчета времени отправления со следующей остановки.

Вторым ключевым алгоритмом является алгоритм создания маршрута, который обеспечивает формирование нового маршрута в системе с валидацией всех входных данных. Алгоритм проверяет корректность введенных данных, включая проверку формата и полноты информации, проверяет условие различия начальной и конечной остановок для обеспечения логической целостности маршрута, проверяет уникальность номера маршрута в системе для предотвращения дублирования и создает объект маршрута с использованием умных указателей для управления памятью (рисунок 4.6). Алгоритм интегрирует новый маршрут в общую структуру данных системы через механизм команд с поддержкой отмены операций, что обеспечивает возможность отката изменений при необходимости. После создания маршрут становится основой для последующего планирования рейсов и формирования расписания движения транспорта.

Третьим важным алгоритмом является алгоритм поиска маршрутов между остановками, который позволяет найти все маршруты, проходящие через указанные начальную и конечную остановки. Алгоритм анализирует все маршруты в системе, проверяет наличие указанных остановок в последовательности маршрута и определяет направление движения, обеспечивая корректный поиск прямых маршрутов между точками отправления и назначения.

Четвертым ключевым алгоритмом является алгоритм планирования поездок с пересадками на основе поиска в ширину (BFS), который позволяет найти оптимальные маршруты между остановками с учетом необходимости пересадок. Алгоритм строит граф связности остановок через маршруты, выполняет поиск в ширину для нахождения путей с минимальным количеством пересадок и формирует полное описание поездки, включая все необходимые пересадки, время отправления и прибытия на каждой остановке.

Дополнительно система поддерживает алгоритм поиска самого быстрого маршрута и алгоритм поиска маршрута с минимальными пересадками, что обеспечивает гибкость в планировании поездок для различных потребностей пассажиров.

Все алгоритмы реализованы с использованием паттерна Strategy, что обеспечивает возможность легкой замены и расширения функциональности без изменения основной структуры системы. Система также включает модуль управления расписанием водителей, который контролирует рабочее время водителей и проверяет соответствие расписания трудовому законодательству (максимум 12 часов работы в день).

Программа обеспечивает сохранение всех данных в файлы при завершении работы и автоматическую загрузку данных при запуске, что гарантирует сохранность информации между сеансами работы. Система поддерживает аутентификацию администраторов для обеспечения безопасности данных и разграничения прав доступа между гостевым и административным режимами.
